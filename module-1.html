    Q-9. How is Cell Padding different from Cell Spacing?

    Cell padding and cell spacing are two properties used in HTML to control the spacing and layout of tables.

    Cell padding refers to the amount of space between the content of a table cell and its border. When cell padding is set to a non-zero value, there will be extra space between the content of the cell and the cell's border. Cell padding can be set using the cellpadding attribute in the table tag, or with CSS.

    In summary, cell padding controls the space between the content of a table cell and its border, while cell spacing controls the space between adjacent cells in a table. It's worth noting that both cell padding and cell spacing can be set to zero (or omitted entirely) if no extra spacing is desired.

    Q-11. What is the difference between a block-level element and an inline element? 

    In HTML and CSS, elements can be classified as either block-level or inline. The main difference between these two types of elements is how they are displayed on a web page and how they interact with other elements.

    A block-level element takes up the entire width of its parent container and creates a new line after it. This means that any content that comes after a block-level element will be pushed to a new line. Block-level elements include headings (h1-h6), paragraphs (p), divs, and lists (ul, ol). Block-level elements are used for larger structural pieces of content on a web page.

    An inline element, on the other hand, does not create a new line after it and only takes up as much width as necessary to display its content. This means that multiple inline elements can appear on the same line. Inline elements include links (a), images (img), and spans. Inline elements are often used for smaller pieces of content within a block-level element, such as text or icons.

    Q-16. How are active links different from normal links? 

    Active links are different from normal links in that they represent a link that is currently being interacted with by the user. This could be a link that the user is hovering over with their mouse or a link that the user has clicked on and is currently being directed to.

    Normal links, on the other hand, represent links that are not currently being interacted with by the user. They are typically displayed with an underline and a default color, and when the user hovers over them, the color may change or the underline may become thicker.

    Active links can be styled differently from normal links using CSS. For example, you can change the color, font, and background of an active link to indicate to the user that they are currently interacting with it. This can help to improve the user experience by making it easier for the user to identify which links they have already clicked on or are about to click on.

    Here's an example of how to style active links using CSS:

    a:link {
    color: blue;
    }

    color: purple;

    a:hover {
    color: red;

    SVG stands for Scalable Vector Graphics. It is a vector image format that is used to display images and graphics on the web. Unlike bitmap images such as JPEGs or PNGs, which are made up of a fixed number of pixels, SVG images are based on mathematical equations, which means they can be scaled to any size without losing quality.

    SVG images are created using XML markup language, which allows them to be easily edited and manipulated. They can be created using a variety of tools, including Adobe Illustrator, Inkscape, and Sketch.

    Q-19. What is difference between HTML and XHTML?

    HTML (Hypertext Markup Language) and XHTML (Extensible Hypertext Markup Language) are both markup languages used for creating web pages. While they share many similarities, there are a few key differences between the two:

    Syntax: The syntax of XHTML is stricter than HTML. XHTML requires all tags to be properly closed and all attributes to be quoted. In HTML, some tags can be left unclosed and attributes don't always need to be quoted.

    Document structure: XHTML requires a document to be well-formed XML, which means that it must have a root element, all elements must be properly nested, and all attributes must have values. HTML, on the other hand, has a looser document structure and doesn't require these strict XML rules.

    Compatibility: XHTML is designed to be compatible with XML tools and technologies, which means that it can be easily integrated with other XML-based applications. HTML, on the other hand, is not as easily integrated with other XML-based applications.

    Browser support: Some older web browsers may not fully support XHTML, whereas HTML is widely supported by all browsers.

    Overall, XHTML is seen as a more structured and strict version of HTML, while HTML is a more flexible and forgiving language. However, HTML5 has largely replaced XHTML as the preferred markup language for web pages.
